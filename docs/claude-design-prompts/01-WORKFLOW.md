# 01 — Workflow: как делать слайды в Claude Design

**TL;DR:** одна сессия Claude Design = один слайд. Между сессиями накапливаем эталоны. К слайду 11 у вас будет 10 готовых слайдов как референс — единый стиль гарантирован.

---

## Подготовка (один раз, перед стартом)

1. **Снять скриншоты ваших 3 существующих слайдов** из `markichyt.github.io/consultantUA/`:
   - `ref-slide-01-welcome.png`
   - `ref-slide-02-profile.png`
   - `ref-slide-03-leads.png`

   Эти 3 файла — ваш «эталонный набор». Будете прикладывать в каждую новую сессию.

2. **Снять 4 скриншота квиза** (нужны только для слайда 7):
   - Откройте `markichyt.github.io/-/` в Chrome DevTools, режим iPhone 14 Pro
   - Снимите экран на каждом из этих шагов:
     - `screenshot-01-contact.png` — первый слайд (имя/email/телефон/фото)
     - `screenshot-02-wowroi.png` — ROI-слайдер «Meet your future on ConsultantLM»
     - `screenshot-03-assessment.png` — оценка Low/Normal/Moderate/High
     - `screenshot-04-pricing.png` — карусель тарифов Base/Pro/Premium

3. **Открыть Finder** в `lastQuiz/` — оттуда будете тянуть видео/фото в Claude Design (см. список ассетов в каждом slide-*.md).

## Цикл по каждому слайду

### Шаг 1. Открыть новую сессию Claude Design
Каждый слайд = чистая сессия. Не пытайтесь делать всё в одной.

### Шаг 2. Скопировать промпт
Открыть `slide-NN-*.md` для нужного слайда. Внутри файла лежит **готовый блок** (с уже встроенным Design System Header сверху). Cmd+A → Cmd+C → Cmd+V в Claude Design.

### Шаг 3. Приложить референсы
Drag-and-drop в сессию Claude Design:
- `ref-slide-01-welcome.png`
- `ref-slide-02-profile.png`
- `ref-slide-03-leads.png`
- **+ все готовые слайды, которые уже сделали** (см. шаг 6 ниже)

### Шаг 4. Приложить ассеты этого слайда
В каждом slide-*.md есть секция «🎯 Ассеты для прикрепления» с полными путями. Перетащите эти файлы из Finder в Claude Design.

### Шаг 5. Сгенерировать. Доработать в чате
Claude Design сделает первую версию. Правьте итеративно:
- «Сделай фон чуть темнее»
- «Увеличь число $14B вдвое»
- «Уменьши логотип, он отвлекает»
- «Замени иконку в карточке 2 на пиктограмму shield»

Не переходите к следующему слайду, пока этот не доведён до удовлетворительного состояния.

### Шаг 6. Сохранить → добавить в эталоны
Когда слайд готов:
- Снимите скрин финального результата: `ref-slide-NN-done.png`
- Положите в ту же папку с референсами
- В следующей сессии прикладываете этот скрин **тоже** — он станет дополнительным якорем стиля

## Порядок прохождения слайдов

Рекомендую делать в этом порядке (не по номерам), чтобы быстро получить визуальный «якорь»:

1. **Слайд 9 — Business Model** первым.
   Почему: самый «инфографически плотный» — диаграммы, тарифные карточки, бары. Если получится — это стилевой бенчмарк для остальных.
2. **Слайд 8 — Market** — второй (тоже инфографика, проще).
3. **Слайд 2 — Problem** — третий.
4. **Слайд 3 — Solution** — четвёртый.
5. **Слайд 6 — AI Content + Avatar** (тёмный вариант).
6. **Слайд 10 — Defensibility** (тоже тёмный).
7. **Слайд 7 — Quiz** (требует подготовки 4 скринов квиза, поэтому ближе к концу).
8. **Слайд 11 — Team & Ask** последним (требует ваши фото и финальные цифры).

## Если что-то идёт не так

| Симптом | Решение |
|---|---|
| Claude Design игнорирует палитру | Прямо в чате: «Use only the exact hex colors from the design system block: #faf9f5, #0A1118, #1397D6, #56C2F0. Replace all other colors.» |
| Шрифт не Outfit/Inter | «Replace all fonts with Outfit (headings) and Inter (body). Confirm via Google Fonts.» |
| Слайд получился слишком пёстрым | «Reduce visual density. Remove decorative elements. Keep only the data and one accent color per zone.» |
| Анимация резкая | «All entry animations should be 240ms ease-out with 120ms stagger. No bounce, no scale-up.» |
| Слайд не похож на референсы | Откатиться, переприложить все эталоны, добавить: «Critical: match the layout grid, card border-radius, and shadow style from the reference screenshots exactly.» |

## Финальная проверка перед сдачей инвестору

После всех 11 слайдов разложите их рядом (в одной презентации или отдельно как PNG). Проверьте 5 пунктов:
- Один и тот же оттенок cyan везде
- Логотип в одном и том же месте
- Все заголовки одной семьи шрифтов и одного размера
- Карточки одного радиуса
- Никакого «свежего» цвета не появилось (фиолетового, бирюзового без основы)

Если хоть один пункт нарушен — попросите Claude Design в новой сессии «normalize this slide to match the rest of the deck». Готово.
